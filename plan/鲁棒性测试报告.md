> **版本**: v1.0

## 1. 测试概述

MarkEye 基础功能开发完成后，本次鲁棒性测试对系统全部功能模块进行逐项验证，覆盖检测算法、Web REST API、WebSocket、Web UI、CLI、配置管理、Modbus IO 及跨平台部署共 **8 大测试域**。

测试分为两类：

| 类型 | 说明 | 本次状态 |
|------|------|----------|
| **自动化** | `pytest` 单元/集成测试、REST API 冒烟、CLI 冒烟 | 已在 Windows 开发机执行 |
| **人工复现** | Web UI 交互、真实相机/Modbus 硬件、Ubuntu 产线部署 | 报告中提供操作步骤，结果栏待填 |

---

## 2. 测试计划

### 2.1 目标与范围

| 目标 | 说明 |
|------|------|
| 功能完整性 | 验证 README / UI 设计稿描述的全部功能是否可用 |
| 鲁棒性 | 记录异常路径、边界条件、跨平台差异 |
| 可复现性 | 每项 FAIL 附复现步骤，供后续人工确认后修改代码 |

**在范围内**：检测管线、Web 服务、向导配置、双路相机逻辑、IO 映射、多程序管理、CLI 调参、启动脚本。  
**不在范围内**：印记加工设备控制、第三方 PLC 程序、网络防火墙策略、硬件选型评估。

### 2.2 测试环境

| 环境 | OS | Python | 用途 | 本次执行 |
|------|-----|--------|------|----------|
| 开发机 | Windows 10 | 3.12.9 | pytest、API 冒烟、CLI、UI 手测 | 已执行自动化部分 |
| 产线机 | Ubuntu 24.04.4 LTS | ≥3.10 | V4L2 相机、kiosk、实机 IO | **待人工复现** |

**前置条件**：
- 虚拟环境已创建：`.venv\Scripts\Activate.ps1`（Windows）/ `source .venv/bin/activate`（Linux）
- 依赖已安装：`pip install -r requirements.txt`
- Web 服务：`python -m src.web_server`（端口 8080）

### 2.3 测试方法与判定标准

| 方法 | 工具 | 通过标准 |
|------|------|----------|
| 单元/集成测试 | `pytest tests/ -v` | 全部用例 exit code 0 |
| API 冒烟 | `Invoke-WebRequest` / curl | HTTP 2xx 且响应体符合契约 |
| CLI 冒烟 | `python -m src.main` | 命令正常退出（检测 FAIL 属业务判定，非崩溃） |
| UI 手测 | 浏览器 `http://localhost:8080` | 界面操作符合预期，无 JS 报错 |
| 硬件联调 | 实机相机 / Modbus | 信号时序与 README 一致 |

**状态标注**：

| 状态 | 含义 |
|------|------|
| **PASS** | 实测通过 |
| **FAIL** | 实测未通过，需修复 |
| **待人工复现** | 需特定硬件或双平台环境，本次未测 |
| **未测** | 环境缺失或时间不足 |

### 2.4 不在范围内

- 性能压测（100 次连续触发）仅在产线机执行
- 真实产品样品 OK/NG 准确率评估（需工艺样品）
- 前端 `config-editor.js`（遗留骨架，未接入 app.js）
